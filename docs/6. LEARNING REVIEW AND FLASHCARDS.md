# 6. Learning Review & Flashcards: Active Recall Interface

## Overview

Feature 6 extends the Isomorphism Engine UI with two new modes for active learning:

1. **Review Tab**: Timeline view of all learnings for quick browsing and rediscovery
2. **Flashcards Tab**: Simple questionâ†’answer flip cards for active recall practice

**Key Design Principle:** This feature leverages the existing learnings database and React UI. We add tabs to the Isomorphism Engine rather than building a separate app, keeping the "disposable wrapper" architecture intact.

---

## The Problem: Passive Knowledge Accumulation

We've built a system that:

- Extracts high-quality learnings from conversations
- Enables semantic search to find relevant past knowledge
- Bridges new concepts with existing learnings (Isomorphism Engine)

**But there's a gap:** The system is still _reactive_ rather than _proactive_:

- We only engage with learnings when we have a specific question
- We don't regularly review what we've learned
- We don't practice active recall (the most effective learning technique)

**The Risk:** Knowledge atrophies. We forget learnings we don't actively revisit.

---

// Existing route
app.post('/api/explain', ...)

app.get('/api/learnings', async (req, res) => {
  const limit = parseInt(req.query.limit as string) || 20
  const offset = parseInt(req.query.offset as string) || 0

  // Query learnings sorted by createdAt DESC
  const learnings = await db
    .select()
    .from(learningsTable)
    .orderBy(desc(learningsTable.createdAt))
    .limit(limit)
    .offset(offset)

  const total = await db
    .select({ count: count() })
    .from(learningsTable)

  res.json({
    learnings,
    total: total[0].count,
    hasMore: offset + limit < total[0].count
  })
})

app.get('/api/learnings/random', async (req, res) => {
  // SQL: SELECT * FROM learnings ORDER BY RANDOM() LIMIT 1
  const rawDb = getRawDb(db)
  const learning = rawDb
    .prepare('SELECT * FROM learnings ORDER BY RANDOM() LIMIT 1')
    .get()

  res.json({ learning })
})

### ReviewView Component

```typescript
export function ReviewView() {
  const [learnings, setLearnings] = useState<Learning[]>([]);
  const [loading, setLoading] = useState(true);
  const [offset, setOffset] = useState(0);
  const [hasMore, setHasMore] = useState(true);

  useEffect(() => {
    fetchLearnings();
  }, []);

  const fetchLearnings = async () => {
    setLoading(true);
    try {
      const response = await fetch(
        `http://localhost:3001/api/learnings?limit=20&offset=${offset}`
      );
      const data = await response.json();

      setLearnings((prev) => [...prev, ...data.learnings]);
      setHasMore(data.hasMore);
      setOffset((prev) => prev + 20);
    } catch (error) {
      console.error("Failed to fetch learnings:", error);
    } finally {
      setLoading(false);
    }
  };

  return (
    <div className="review-view">
      <h2>Your Learning Timeline</h2>

      <div className="timeline">
        {learnings.map((learning) => (
          <LearningCard key={learning.learningId} learning={learning} />
        ))}
      </div>

      {hasMore && (
        <button onClick={fetchLearnings} disabled={loading}>
          {loading ? "Loading..." : "Load More"}
        </button>
      )}
    </div>
  );
}
```
